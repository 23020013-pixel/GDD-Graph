Project layout: api/main.py (app entry), api/routers/ (endpoint groups), api/models/ (Pydantic), api/db.py (Neo4j driver), api/cache.py (Redis), api/config.py (env vars qua pydantic-settings). Endpoint GET /health trả `{status: "ok", neo4j: "connected"}`.
api/db.py khởi tạo AsyncGraphDatabase.driver(uri, auth=...) ở app startup event, close ở shutdown. Connection pool config max_connection_pool_size=50. Wrapper function async def run_query(cypher, params) -> list[Record]. Test bằng query đơn giản RETURN 1 as one từ endpoint.
File api/models/entities.py định nghĩa: Gene(id, name, aliases, chromosome), Disease(id, name, synonyms, doid), Drug(id, name, inchikey, drugbank_id), Edge(source_id, target_id, relation_type, sources, scores, aggregated_score, last_updated). Dùng Field(..., example=...) cho OpenAPI docs đẹp. Validator custom cho ID format.
api/cache.py khởi tạo redis async client. Integrate slowapi middleware cho rate limit: mặc định 60 req/min per IP, endpoint /path giới hạn 10 req/min (đắt hơn). Custom key function dùng X-Forwarded-For nếu có. Response khi limit: 429 với Retry-After header.
Router api/routers/entity.py. Cypher query dùng parametrized MATCH (n {id: $id}) RETURN n, labels(n) as labels. Response model EntityResponse union Gene/Disease/Drug dựa trên labels. Test 3 ID thật: 1 Gene (ví dụ Gene::5243), 1 Disease (Disease::DOID:1612), 1 Drug (Compound::DB00945). Invalid ID trả 404 với error schema chuẩn.
Router api/routers/disease.py. Query MATCH (d:Disease {id: $id})-[r:ASSOCIATES_DaG]-(g:Gene) RETURN g SKIP $offset LIMIT $limit. Query params: limit (1-100 default 20), offset (≥0 default 0). Response có total_count (dùng query count riêng), items[], has_next. Benchmark với limit=50 target < 500ms.
File api/errors.py định nghĩa APIError(code, message, details). Exception handlers cho: Neo4jError → 503, ValidationError → 422, HTTPException → original code, Exception → 500. Error codes dùng string: ENTITY_NOT_FOUND, INVALID_ID_FORMAT, RATE_LIMIT_EXCEEDED, etc. Document codes trong docs/api_errors.md.
Thư mục tests/api/ với test_health.py, test_entity.py, test_disease.py. Dùng pytest-asyncio + httpx.AsyncClient. Fixture test_client với FastAPI TestApp. Mock Neo4j driver cho unit test, real Neo4j cho integration test (mark @pytest.mark.integration). Coverage target > 70% cho api/routers/.
Thư mục ui/ với app.py (home page) và ui/pages/ (mỗi file là 1 page). Dùng st.navigation API mới. st.set_page_config với title "Hetionet Explorer", layout wide. Sidebar: link 4 page + logo + version number + API health indicator (ping /health mỗi 30s).
Page 1 Entity Search: search box + tabs tìm theo Gene/Disease/Drug + card hiển thị detail. Page 2 Relationship Explorer: 2 input (source, target) + slider max_hop + graph viz area. Page 3 Drug Recommendation: disease picker + top-k slider + table với columns (drug, score, sources). Page 4 Pipeline Dashboard: 4 metric card + timeline chart (placeholder). Mock data JSON trong ui/mocks/.
Dùng streamlit-agraph hiển thị graph 20 node. Config node colors theo label (Gene=xanh, Disease=đỏ, Drug=vàng), node size theo degree. Click node → st.session_state lưu selected node → panel bên phải hiện detail. Thử render tại Page 2 với mock neighborhood.
